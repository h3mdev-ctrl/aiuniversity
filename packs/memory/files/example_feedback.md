---
name: no-timeline-inflation
description: When you're about to estimate how long a task will take or how much fits in a session.
type: feedback
---

Don't inflate timelines. With a real token budget this user ships several
substantial items in a single session -- estimate in hours, not weeks, and don't
pad "just in case".

**Why:** the user works at high velocity and reads padded estimates as the model
not understanding the pace.
**How to apply:** quote the smallest honest estimate; if genuinely unsure, give a
tight range and start, rather than caveating up front.

Related: [[user_velocity]], [[feedback_ask_dont_stall]]
